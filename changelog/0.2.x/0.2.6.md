# 0.2.6 — 2026-06-12

## Added

- **`openstreetmap_geocode` truncation disclosure**: the handler now calls `ctx.enrich.truncated({ shown, cap })`, surfacing `truncated` / `shown` / `cap` enrichment fields so an agent can tell when the result set was capped at the requested `limit`. Adopts the 0.10.x linter rule that flags undisclosed truncation.
- **Explicit server identity**: `createApp()` sets `name` and `title` to `openstreetmap-mcp-server`, forwarded to the `initialize` response and `/.well-known/mcp.json`.
- **`scripts/clean-mcpb.ts`**: post-pack `.mcpb` cleaner — runs `mcpb clean` (dev-dependency prune + manifest validation) and strips dependency-shipped agent docs (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns can't reach. Wired into the `bundle` script.
- **Docker `HEALTHCHECK`**: bun-native `fetch` against `/healthz` (the slim image ships no curl/wget), plus an `org.opencontainers.image.version` OCI label populated from the `APP_VERSION` build arg.

## Changed

- **`scripts/check-framework-antipatterns.ts`**: adds rule 4 — `z.coerce.boolean()` on an env flag (can't be disabled through the environment, since `Boolean("false")` is `true`; use `z.stringbool()`), scoped to `src/` so it fires in consumer config. Matched comment lines are now skipped so JSDoc naming an antipattern isn't reported as a usage.
- **`scripts/lint-packaging.ts`**: gitignore-aware bundle-content checks, post-bundle agent-doc entry assertions (shared `AGENT_DOC_ENTRY` pattern with `clean-mcpb.ts`), and an entrypoint/manifest identity check warning when the `name`/`title` pair is absent.
- **`.mcpbignore`**: patterns root-anchored with a leading `/` so they match only top-level paths.
- **Skills**: vendored skill set synced to mcp-ts-core 0.10.6; `techniques` skill added.
