# 0.2.1 — 2026-05-28

## Added

- **`distance_meters`** on each `query_nearby` result: great-circle (haversine) distance from the query center, rounded to one decimal meter. Elements without a computable center sort last with the field omitted.

## Changed

- **`query_nearby`** and **`query_bbox`** descriptions now state they always return the full OSM tag set and that `extratags` applies only to the Nominatim tools (`geocode` / `reverse` / `lookup`). ([#8](https://github.com/cyanheads/openstreetmap-mcp-server/issues/8))
- **`query_nearby`** description drops the old "results are not sorted by distance" caveat, now reflecting sort-before-truncate behavior.

## Fixed

- **`query_nearby`** returned features in OSM element-ID order with no distance, so `limit` could silently drop the nearest matches while keeping farther ones. Results are now sorted nearest-first before the `limit` slice, so truncation keeps the closest N. ([#7](https://github.com/cyanheads/openstreetmap-mcp-server/issues/7))
