# 0.2.8 — 2026-06-20

Framework maintenance only — no tool or behavior changes.

## Added

- **`scripts/check-dependency-specifiers.ts`** — new devcheck step (`Dependency Specifiers`, flag `--no-dep-specifiers`) that hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` dependency sections and `bun.lock` workspace specifier maps. Catches `bun update --latest` writing a `latest` dist-tag into the lockfile. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **`devcheck.config.json`** — `packaging.pluginManifests: true` enables `lint:packaging` check 10, validating `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` for non-empty descriptions and correct scoped/unscoped identity. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`.codex-plugin/plugin.json`** — populated `interface.longDescription` to satisfy the new plugin-manifest description check.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.6` → `^0.10.9` — `ctx.content` collector for image/audio blocks; Canvas SQL gate classifies `SELECT`-shaped prepare failures as `invalid_sql` with DuckDB binder detail; `OTEL_SERVICE_NAME` seeding docs corrected; `DuckdbProvider.describe()` filter-qualification fix.
- **devcheck scripts re-synced** — `build-changelog.ts`, `devcheck.ts`, and `check-framework-antipatterns.ts` skip git-dependent and changelog-sync checks on a fresh scaffold (before `git init` / before any version files exist); `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree. ([cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
- **Framework skills re-synced** — `api-*`, `field-test`, `git-wrapup`, `orchestrations`, `polish-docs-meta`, `report-issue-local`, `tool-defs-analysis` picked up upstream body and `metadata.version` bumps. ([cyanheads/mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238))
