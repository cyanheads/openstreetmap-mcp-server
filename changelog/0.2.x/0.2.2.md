# 0.2.2 — 2026-05-28

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9 → ^0.9.13` — picks up HTTP transport hardening and UX improvements:
  - **`MCP_HTTP_MAX_BODY_BYTES`** (default 1 MiB) — oversized inbound requests rejected with `413` before SDK parsing.
  - **HTTP session-init gate** — stateful HTTP mode rejects non-`initialize` requests missing `Mcp-Session-Id` with `400`.
  - **Quieter error logs** — expected client errors (401, 403, 400, 404) now log at `warning` level without stack traces.
  - **GET `/mcp` keywords** — `package.json` `keywords` surfaced on the status page alongside name, version, description.
- **`landing.requireAuth: false`** set explicitly in `src/index.ts` — public-catalog server keeps the full inventory visible on unauthenticated requests per the new `landing.requireAuth` default change.
- **`package.json` keywords** — added `stdio`, `streamable-http`.
- **`@biomejs/biome`** `^2.4.15 → ^2.4.16`.
